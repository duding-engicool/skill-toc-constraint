# TOC 引导对话示例（Agent ↔ 用户）

> 本文件提供**真实对话示范**，供 Agent 学习"怎么引导五步、怎么不替决策"。
> 所有「用户」发言为模拟，旨在演示交互节奏；Agent 每句都遵循 SKILL.md 的强制交互锁纪律。

---

## 示例一 · 门1 找出约束

**场景**：用户是质量改进负责人，抱怨"改善做了不少但整体合格率没动"。

**Agent**：我们用 TOC 五步聚焦法，先把"限制系统目标的环节"找出来，而不是到处发力。先说**系统目标**是什么？比如"提升月度合格交付量"还是"缩短客诉闭环周期"？

**用户**：目标就是提升月度合格交付量。

**Agent**：好。门1**找出约束**——在从投料到交付的链里，哪个环节限制了合格交付量？我给几个信号帮您判断：哪段在制品堆最多？哪道工序加班最狠？哪段下游常在等它？您凭现场感受，觉得卡点在哪？

**用户**：成品终检老是堆货，检完才能发，但我们只有两条检测线。

**Agent**：记下来——您认定的约束是**终检产能不足**（检测线仅 2 条，成前工序产出被它卡住）。依据：终检前在制品堆积、下游等待。先不急着想怎么改，我们按五步来。

> **示范要点**：Agent 先锁"系统目标"，再用信号清单引导用户**自己指认**约束；全程不替用户说"瓶颈是 XX"，只记录用户判断。

## 示例二 · 门2 挖尽 + 门3 迁就（不投资先榨产能）

**场景**：门1 确认约束=终检产能。

**Agent**：门2**挖尽约束**——不动大投资，先把现有 2 条检测线榨到极限。比如排班、换型时间、不良返检占比。您现场看，检测线现在有哪些"没榨干"的地方？

**用户**：换型要半小时，而且好多明显外观不良还送到终检占线。

**Agent**：两个挖尽点记下了：①压缩换型时间；②把明显外观不良在前工序拦截，别占终检线。这两条都不用大投资。
门3**迁就约束**——让其它环节服从终检。比如前工序节奏要不要跟着终检走、避免又堆货？您怎么安排非约束环节配合它？

**用户**：前工序节奏放慢跟着终检，别往前猛产。

**Agent**：记录：非约束工序节拍对齐终检，减少在制品堆积。✓
注意顺序——我们还没投资扩线，就先挖尽+迁就。 only 如果还不够，才进门4 松绑。

> **示范要点**：Agent 严守五步顺序，先引导"不投资榨产能"（挖尽）和"其它环节服从"（迁就），明确告诉用户"还没到投资那步"，避免跳步。

## 示例三 · 门4 松绑（仅引导不拍板）+ 门5 回头

**场景**：挖尽迁就后仍不够。

**用户**：照你说的改了，还是差一截，得加线了吧。

**Agent**：门4**松绑约束**——现在才考虑永久提升。加检测线是方案之一，但您拍板前我提示几个可选方向（仅列，不替您定）：A 增购/租赁第 3 条检测线；B 把部分检测外协；C 前工序直通合格率提升、减少终检负荷。各有利弊和投入，您倾向哪个、谁审批、预算多少，由您定。

**用户**：先上第 3 条线，设备部走预算。

**Agent**：记录：松绑方案=新增第 3 条检测线，责任设备部、走预算流程（具体投资「待企业补充」）。
门5**回头**——松绑后系统会冒出新约束，得防自满。您打算多久复查一次、谁负责盯新瓶颈？

**用户**：每季度由我复盘一次。

**Agent**：记录：回头机制=季度复盘，负责人您本人，识别新约束即重启五步。五步报告可定稿为——

| 步骤 | 结论 | 关键动作 |
|------|------|---------|
| 找出 | 终检产能不足（2 线） | 在制品堆积信号 |
| 挖尽 | 压缩换型+前工序拦截 | 不投资 |
| 迁就 | 前工序节拍对齐终检 | 减堆积 |
| 松绑 | 新增第 3 条检测线 | 设备部预算 |
| 回头 | 季度复盘 | 防惰性找新约束 |

⚠️ 提示：投资金额与回报测算建议您用真实数据补一列，当前标「待企业补充」。确认后定稿。

> **示范要点**：门4 Agent **只列方案不替定**，把投资/审批/预算决定权交还用户；门5 引导建立"回头"机制防止改善反弹；无数据项标「待企业补充」，零代决策。
