# TOC约束识别与突破技能（toc-constraint）

> 主色：`#C8102E` ｜ 类型：交互引导式（分析/规划类，强制交互锁）

## 一句话简介
交互引导式运用 TOC 五大聚焦步骤（找出 / 挖尽 / 迁就 / 松绑 / 回头）识别系统约束并制定突破策略，**不替用户决策**。

## 适用角色
运营负责人、质量改进负责人、流程所有者、持续改进推进者。

## 何时使用
- 改善没重点、资源平均用力却整体无提升；
- 想系统识别流程/项目/组织的瓶颈（不限于产线）；
- 局部改善做了但系统产出没变；
- 质量改进想聚焦真正限制合格率/效率的环节。

## 五道强制交互锁（五步聚焦法）
1. **找出**：识别限制系统目标的约束；
2. **挖尽**：不投资先榨干现有约束产能；
3. **迁就**：让其它环节服从并服务约束；
4. **松绑**：仍不足才投资/工艺/外协永久提升；
5. **回头**：松绑后出现新约束，回到步骤1防惰性。

## 产出物
`TOC约束分析报告.md`：五步结构 + 约束依据 + 突破路径 + 回头机制。

## 与参考技能的区别
`TOC瓶颈改善` 聚焦产线/工序数据量化；本技能取其**通用系统视角**（约束可为政策/资源/市场/信息流/组织惯性），重引导决策而非数据机检。

## 设计规范
- 简体中文文档，代码标识符英文；
- 主色 `#C8102E`；
- 防幻觉：无数据项标「待企业补充」；
- 去商业化：无营销人设、无付费引导；
- 强制交互锁：Agent 绝不替用户指认约束或决定方案。
